## Codebase Patterns
- Docs navigation derives from filesystem traversal: directories are processed before files, all alphabetically, to keep deterministic order.
- Synthetic section nodes are title-cased from slug segments; markdown pages override titles/summaries from frontmatter and carry isPage markers for nav flattening.
- Vitest runs via `npm test`; `npm run build` performs Next.js build with type-checking.
- Components that use Next.js Link are mocked to plain anchors in tests; keep React imported when server-rendering with Vitest.
- Docs pages consume the doc tree and pass `slug.join('/')` as the active slug for sidebar highlighting.
- Docs layout lives in `app/docs/layout` and reads `params.slug` (Promise) to set active sidebar links; doc content renders inside `.docs-content` while the sidebar remains sticky.
- Prev/next navigation pulls neighbors from `getDocNavList()` via the `DocsPrevNext` component, which takes the current slug string; ensure React is imported for server-rendered docs pages/components.
