## Codebase Patterns
- Backstage catalog model and validators live in `lib/catalog-shared.ts`; entity refs normalize to `Kind:namespace/name`.
- Relationship computation lives in `lib/catalog-core.ts` (deriveCatalogRelationships, attachCatalogRelationships).
- `lib/content.ts` is the thin integration layer: re-exports catalog-shared, catalog-loader, catalog-core, wraps getCatalogEntities/getCatalogContent.
- `catalog-core.ts` is authoritative for CatalogFacets, CatalogFilters, CatalogEntityWithRelationships, filterCatalogEntities, groupCatalogEntities.
- Facets use entity refs (e.g., `System:platform/dev-portal`) not bare names for systems/domains.
- Filter matching normalizes entity refs for system/domain comparisons via `normalizeRef`.
- Tests run with Vitest via `npm test`; type safety is checked with `npm run typecheck`.
- Test isolation: run all tests together, not individually, to catch state issues.
