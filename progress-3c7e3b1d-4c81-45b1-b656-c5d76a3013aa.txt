## Codebase Patterns
- Backstage catalog model lives in `lib/catalog-shared.ts` (types, validators, ref utils) and `lib/catalog-core.ts` (relationship derivation, facets, filters, grouping).
- `lib/catalog-loader.ts` scans `content/catalog` for `catalog-info.yaml` files, validates via `validateCatalogEntityEnvelope`, and derives slugs from entityRefs.
- `lib/content.ts` re-exports everything and provides `getCatalogEntities`, `getCatalogContent`, `getCatalogFacets`, and doc tree helpers.
- Tests run with Vitest (`npm test`); typecheck with `npm run typecheck` (tsc --noEmit). vitest.config.ts requires `@vitejs/plugin-react` plugin and `include` both `tests/**` and `app/**` patterns.
- Entity refs normalize to `Kind:namespace/name`; kinds use PascalCase (Component, API, System, Resource, Domain, Location). Default namespace is `default`. `parseEntityRef` accepts optional `{ defaultKind, defaultNamespace }` context when resolving relative refs.
- `normalizeTarget(raw, fallbackKind)` resolves `namespace/name` patterns by prepending the fallback kind.
- `filterCatalogEntities` supports owner, tag, tags, kind, namespace, system, domain filters (AND semantics).
- `getCatalogFacets` returns owners, tags, kinds, namespaces, systems (as entityRefs), domains (as entityRefs).
- `lib/catalog-core.ts` `exports catalogKindOrder` from catalog-shared. Do not re-export `CatalogKind` as value — it's a type only.
